# Speech kb
This is just for testing an simple idea for __fast writing documents with speech to text__.

Google gives us excellent tools for speech to text.  And I hope those tools are coming to Google Chrome on Android too.  Gboard on Android is very good, but I currently lack one feature: When you use speech to text text you have to give also the commands as voice input.  This possibility is of course very good for many people, but not very good if you want to give a command fast and reliably.  (It is actually often quite bad if you are not using English.)

So in this little app I have tried to mix voice input with some basic commands on the screen.

### What is missing in Chrome on Android?
The missing part there, that is crucial to this app, is __"on device speech to text"__. (This is actually already available today, 2026-07-22, in Chrome on Windows 11, but not on Android yet.) It is _an experimental feature that is not yet standardized_, but I guess Google will soon make it more widely available, maybe when Pixel 11 is ready?
On device speech to text is much faster and therefore much more fun to use in an app like this!
